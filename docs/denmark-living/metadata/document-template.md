## Template Usage Notes

**This section should be removed when creating actual documents**

### Section Guidelines

1. **Overview**: Must include key topic keywords in first sentence for RAG optimization
2. **Key Information**: Quick reference box - keep concise
3. **Procedural Sections**: Use H2 for main sections, H3 for subsections
4. **Steps**: Always use numbered lists for sequential procedures
5. **Common Questions**: Address frequently asked questions identified in requirements
6. **Deadlines**: Highlight time-sensitive information prominently
7. **Troubleshooting**: Include common issues and edge cases
8. **Contact Information**: Provide multiple ways to get help
9. **Related Topics**: Use relative markdown links
10. **Official Resources**: Always include source attribution

### Writing Style

- Use clear, simple English
- Define Danish terms on first use: **CPR-nummer** (civil registration number)
- Be specific and actionable
- Include examples where helpful
- Use bullet points and formatting for readability
- Keep sections under 800 tokens for optimal chunking

### Metadata

- Update `last_updated` date when content changes
- Include all relevant keywords for search optimization
- Specify correct audience (EU Citizens, Non-EU Citizens, or All Residents)
- Always include `source_url` from borger.dk or other official source

### Cross-References

- Link to related topics using relative paths: `../category/document.md`
- Provide context in link text: "See [CPR Number guide](../arrival-process/cpr-number.md) for registration information"
- Ensure all links point to existing or planned documents

### Danish Terms

- Bold Danish terms on first use
- Provide English translation in parentheses immediately after
- Add term to glossary.md if not already present
- Use consistent terminology throughout documentation
